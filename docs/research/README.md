# Research Knowledge Base

Drop all research Markdown files here.

## Suggested structure

Create subfolders by area, for example:

- `docs/research/ai-writing/`
- `docs/research/coding-assistants/`
- `docs/research/automation/`
- `docs/research/marketing/`

## File naming

Use clear, sortable names:

- `YYYY-MM-DD-topic-source.md`

Example:

- `2026-03-09-best-coding-assistants-comparison.md`

## Optional metadata block (recommended)

```md
---
area: coding-assistants
question_intent: "best tool for coding"
source_type: research-notes
updated_at: 2026-03-09
---
```

This makes it easier to retrieve and use the notes for real user questions later.
